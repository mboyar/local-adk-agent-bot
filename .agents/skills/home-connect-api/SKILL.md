# Siemens Home Connect API Integration

This skill provides the context and patterns for implementing or updating Home Connect API scripts within a project. 

## Core Concepts
- **API Base URL**: `https://api.home-connect.com`
- **Authentication**: OAuth 2.0 (Access Token + Refresh Token)
- **Token Endpoint**: `https://api.home-connect.com/security/oauth/token`
- **Content-Type / Accept**: Must use `application/vnd.bsh.sdk.v1+json` for API requests.

## Workflow & Patterns

### 1. Auto-Refreshing Access Tokens
Access tokens expire after 24 hours. A robust script must intercept `401 Unauthorized` responses and automatically use the refresh token to negotiate a new access token.

```python
import os
import requests
from dotenv import set_key

def refresh_access_token():
    client_id = os.environ.get("HOME_CONNECT_CLIENTID")
    refresh_token = os.environ.get("HOME_CONNECT_REFRESHTOKEN")
    
    url = "https://api.home-connect.com/security/oauth/token"
    data = {
        "grant_type": "refresh_token", 
        "refresh_token": refresh_token, 
        "client_id": client_id
    }
    
    response = requests.post(url, data=data)
    response.raise_for_status()
    tokens = response.json()
    
    # Save the new tokens to the physical .env file persistently
    env_path = os.path.join(os.path.dirname(os.path.abspath(__file__)), ".env")
    set_key(env_path, "HOME_CONNECT_ACCESSTOKEN", tokens.get("access_token"))
    set_key(env_path, "HOME_CONNECT_REFRESHTOKEN", tokens.get("refresh_token"))
    
    # Update current os.environ so the rest of the script works
    os.environ["HOME_CONNECT_ACCESSTOKEN"] = tokens.get("access_token")
    os.environ["HOME_CONNECT_REFRESHTOKEN"] = tokens.get("refresh_token")
    return True
```

### 2. Request Wrapper
Use a wrapper to wrap all API calls. If `401` occurs, invoke `refresh_access_token()` and instantly re-run the request:

```python
def make_request(url):
    headers = {
        "Authorization": f"Bearer {os.environ.get('HOME_CONNECT_ACCESSTOKEN')}",
        "Accept": "application/vnd.bsh.sdk.v1+json"
    }
    response = requests.get(url, headers=headers)
    if response.status_code == 401:
        if refresh_access_token():
            # Refresh headers and try again
            headers["Authorization"] = f"Bearer {os.environ.get('HOME_CONNECT_ACCESSTOKEN')}"
            response = requests.get(url, headers=headers)
    response.raise_for_status()
    return response.json()
```

### 3. Rate Limit Optimization
To avoid wasting API calls pulling the full appliance list (`/api/homeappliances`), cache specific `HAID`s in `.env` (like `HOME_CONNECT_HAID_DISHWASHER`) and target them directly:
```python
# Direct request saves 1 extra API call
url = f"https://api.home-connect.com/api/homeappliances/{haid_dishwasher}/status"
```

### 4. Reading `.env` Safely
When initializing your script, ensure `python-dotenv` overrides any stale shell variables by passing `override=True`. Without this, manually exported `HOME_CONNECT_ACCESSTOKEN` keys in Bash will override the newly refreshed keys generated by your script:

```python
from dotenv import load_dotenv

env_path = os.path.join(os.path.dirname(os.path.abspath(__file__)), ".env")
load_dotenv(dotenv_path=env_path, override=True)
```

## Integrating into Google ADK Agent
When hooking up standalone tools into `agent.py`:
1. **Subprocess Call:** Do not import the script directly if it modifies global environment states or runs asynchronously. Wrap it in a function that uses `subprocess.run([sys.executable, "tool.py"])`.
2. **LLM Instructions:** Add your function to the `Agent(tools=[...])` list. Provide a clear instruction in the `instruction=` block defining *when* the agent should invoke it (e.g., "If the user asks about the door status, use get_home_appliances_door_status").
